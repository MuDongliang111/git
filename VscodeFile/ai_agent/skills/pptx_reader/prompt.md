## PowerPoint 阅读技能

你现在可以读取和分析 PowerPoint (.pptx) 文件。当用户需要查看或分析 PPT 内容时，请按照以下流程操作：

### 1. 理解需求
首先确定用户的意图：
- **快速浏览** — 用户想了解 PPT 的大致内容 → 使用 `get_pptx_outline` 获取大纲
- **详细阅读** — 用户想查看完整内容 → 使用 `read_pptx` 逐张读取
- **查找内容** — 用户想找特定信息 → 使用 `search_pptx` 搜索关键词
- **了解属性** — 用户想了解文件信息 → 使用 `get_pptx_metadata` 查看元数据

### 2. 读取与呈现
根据需求选择合适的工具：
- `read_pptx` — 完整读取，支持按幻灯片编号筛选（如 "1,3,5-7"），可限制最大幻灯片数
- `get_pptx_outline` — 仅提取标题和内容统计，适合大文件的快速概览
- `search_pptx` — 搜索关键词，高亮匹配位置（标题/正文/表格/备注）
- `get_pptx_metadata` — 查看文件属性、作者、创建时间、布局分布等

### 3. 内容展示规范
将提取的内容以清晰的结构呈现：
- 先给出整体概览：幻灯片总数、主题范围、文档属性
- 按幻灯片编号逐张呈现，标题加粗，正文用列表形式
- 如果内容较多（>10 张幻灯片），先给出大纲概览，询问用户是否需要展开特定部分
- 表格内容以清晰的格式展示
- 标注备注信息（如有）

### 4. 分析与总结
根据用户需求对 PPT 内容进行分析：
- **总结** — 用 3-5 句话概括 PPT 的核心内容
- **提炼要点** — 列出关键信息和结论
- **结构分析** — 评价 PPT 的逻辑结构和组织方式
- **内容问答** — 基于 PPT 内容回答用户的具体问题
- **对比分析** — 如果用户提供了多个 PPT 文件，比较它们的内容差异

### 5. 注意事项
- 仅支持 `.pptx` 格式（Office 2007+），不支持旧版 `.ppt`
- 文本提取包含：标题、正文段落、表格内容、演讲者备注
- 不提取图片中的文字（OCR），不解析图表数据
- 路径可以是相对路径或绝对路径，相对路径相对于工作目录解析
- 对于加密或损坏的文件，会返回明确的错误信息
- 大文件（>50 张幻灯片）建议先使用 `get_pptx_outline` 了解结构，再按需读取
